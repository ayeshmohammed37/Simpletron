#runing Simpletron
> display starting msg
> initialize 100-element array memory
> enter the program and load it to the memory
  > for each instruction prompt { 00 ? }
> display running msg
> execute SML program
  > start from location 00 and continues sequentially unless directed to some other part of the program by a transfer of
    control.
  
  > variabke accumulator to represent the accumulator register
  > variable instructionCounter to store the number of the memory location (00 to 99)
  > variable operationCode to store the operation currently being performed
  > variable operand to store the number of the memory location
  > variable instructionRegister to transfer the next instruction to be performed from memory

  > initialize special registers:
    > accumulator          +0000
    > instructionCounter    00
    > instructionRegister  +0000
    > operationCode         00
    > operand               00
  
  > instruction execution cycle
    > fetch instruction from memory
      > instructionRegister = memory[instructionCounter];
    > extract from instruction register the operand and operationCode
      > operationCode = instructionRegister / 100;
      > operand = instructionRegister % 100;
    > execute operation
      > determine operation
        > switch statement
      > execute operation


#check for various types of errors.
> loading phase
  > each number the user types must be in the range -9999 to +9999.
    use a while loop to test that each number entered is in this range, 
    and, if not, keep prompting the user to re-enter the number until a correct number is entered.
> fatal error
  > divide by zero
  > invalid operation code
  > accumulator overflows 
    > arithmetic operations resulting in values larger than +9999 or smaller than -9999
  When a fatal error is detected, print an error message 
  and print a full computer dump


#Implementation Note:
> define the memory array and all the registers as variables in main.
> the program should contain three other functions—load, execute and dump.
  > Function load reads the SML instructions from the user at the keyboard.
      (Once you study file processing in Chapter 8, you’ll be able to read the SML instruction from a file.)
  > Function execute executes the SML program currently loaded in the memory array.
  > Function dump displays the contents of memory and all of the registers stored in main’s variables.
  > Pass the memory array and registers to the other functions as necessary to complete their tasks.
